Studio

While Studio is active its manifest declares chrome.menu = "substitute": the host yields the portal side menu and renders the app's own items.

⟨ federated Studio remote renders here ⟩
The portal menu (Organizations / Billing / Profile …) is hidden while substituted. The host always keeps a non-removable Return to portal control so the user is never trapped — supplied by the host, not the app. On exit the host restores its own chrome. Menu items come from manifest.chrome.items.
